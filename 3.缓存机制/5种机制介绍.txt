Cache First 缓存策略
  含义：
    1.若有缓存，则直接使用缓存，不发起请求
    2.若无缓存，发起请求，更新缓存
  场景：适合构建完全离线优先内容，几乎不变化的资源可以采用Cache First

Cache Only 缓存策略
  含义：
    1.若有缓存，则直接使用缓存，不发起请求
    2.若无缓存，返回错误
  场景：基本不使用

Network First 缓存策略
  含义：
    1.若有缓存，依然请求，优先使用请求返回，请求失败，降级到缓存
    2.若无缓存，请求接口成功后使用返回内容并更新缓存
  场景：容灾场景适用，接口兜底

Network Only
  含义：
    1.如果你需要从网络中完成特定请求，则首先要使用网络策略

StaleWhileRevalidate 缓存策略
  含义：
    1.若有缓存，直接使用缓存，同时发起请求，请求成功后更新缓存
    2.若无缓存，发起请求，请求成功后使用返回内容并更新缓存
  场景：通常用于频繁更新，但不需要最新版本的资源
